git Branch 사용 방법

Branch: 서로 다른 환경에서 서로 다른 목적으로 개발할 수 있도록 함

Master Branch: Merge가 되기전까지
항상 안정화되도록 만들어야함

통합 브랜치: 배포가 가능한 수준의 브랜치로 일반적으로 마스터 브랜치
토픽 브랜치: 특정한 기능을 위해 만들어진 브랜치로 일반적으로 마스터 브랜치
이외의 다른 브랜치

git branch [브랜치 이름]: 브랜치 생성

git checkout [브랜치 이름]:  다른 브랜치로 접속

1.git add -> 수정된 파일 추가
2. git commit 
3. git merge [브랜치 이름] -> 기존꺼랑 합치기
4. git push -> Github에 업로드


git branch -d [브런치 이름]: 브런치 삭제
