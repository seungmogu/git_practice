Github(오픈 소스)
하나의 프로젝트를 여러사람들과 함께 협업해서 관리하기위해 만들어짐

오픈소스를 사용 하는 이유: 
팀원과 프로젝트를 함께 관리하는 것 뿐만 아니라
다른 개발자들도 사용해보고 이를 평가하여 개선가능

Github 장점
중앙서버 없이 협업 가능 -> 분산형 협업 도구
동시에 작업 가능 -> 효율적인 개발 가능
비선형적 개발: 프로젝트의 가지치기 가능
변경이력 보장: 작업 내역이 모두 별도의 영역에서 관리되어 문제가 생기더라도 불러오기 가능
포트폴리오 관리에 용이

Git 동작원리
Working Directory: 작업할 파일이 있는 디렉토리
Staging Area: 커밋을 수행할 파일들이 올라가는 영역
Git Directory: 작업한 영역을 반영하는 곳

git ADD Commit Push 명령어: 깃허브에 반영
git Fetch Merge 명령어 -> git Pull: 우리 컴퓨터로 데이터 반영

Repository(Ripo): 실제 소스코드가 담겨있으면서 커밋 내역등 모든 작업이력이
담긴 공간 

gitignore: 올리지 않고자하는 파일(데이터베이스)
license: 오픈 소스 라이센스 중에서 어떤걸 채택하고 있는지 알리기 위함

우리 컴퓨터: Local Repository
우리의 깃허브 주소: Remote Repository
